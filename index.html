Welcome to the go-ssh page

If you work in UNIX environments as a Developer, a Systems Administrator, or a Systems Engineer you've probably already encountered the frustration of trying to keep track of all of the hosts, credentials, and users that you're required to connect to in performing your day-to-day tasks. This is a particularly large pain if you happen to be working freelance, or as a consultant in a role that requires you regularly connect to large numbers of hosts, often for different customers, and have to dig around in your documentation to confirm hostnames, username, SSH private key files, etc.

Go-SSH is intended to address that kind of frustration by providing a command-line utility that allows users to manage SSH connections to a large number of disparate hosts, requiring differing users and keys in a simple, efficient, and hassle-free manner.

What Go-SSH is:

Go-SSH is a simple and, at present very hacked together, Python wrapper around the standard OS ssh, and SCP commands on most unix, or unix-like operating systems.

  • It does launch SSH connections based on a connection name. Each connection definition includes:
    • The target hostname or IP address
    • The remote username to connect as
    • The location of the SSH private key file to authenticate with.
  • It does provide functionality to add, modify, and delete connection profiles as part of the single "go" command. No need to edit configuration files by hand.
  • It is written to ensure that there are no extra library requirements or packages to install. As long as your environment has a Python install and ssh and scp installed - it will work

What Go-SSH is not:

  • Go-SSH is not intended to be a replacement for your OS installed ssh and scp binaries; There are plenty of SSH clients out there. Creating another one wouldn't have solved the problem we were aiming at - to have a simple, intuitive and above all easy utility to manage all our SSH connections in one spot.
  • It is also not a gui wrapper to SSH with connection management built in. There are plenty of solutions out there that do this for Windows, Mac, and UNIX environments

Where we're at with Go-SSH

To be frank - we've only just started. There will be a proportion of the features listed ( below ) that at the time of writing are either planned or only partly implemented.

Current features

  • command-line management of your connections using the 'go' utility ( add, modify and remove )
  • Simple listing of all current connections ( "go -l" )
  • Reliable start and exit of SSH sessions to your configured connections ( "go CONNECTION_NAME" )

Planned features

  • Simple file push feature. Push a file over SCP. ( "go NAME SRC REMOTE_DIR" )
  • Simple file pull feature. Pull a file over SCP. ( "go NAME REMOTE_SRC DEST_DIR" )
  • Simple dir list feature. Retrieve "ls -l" output for DIR ( "go NAME list DIR" )
  • Simple ps output feature. Retrieve "ps -ef" output from NAME ( "go NAME ps" )
  • Simple netstat output feature. Retrieve "netstat -an" output from NAME ( "go NAME netstat" )

As you can see - most of these features are geared towards users who are managing, or supporting hosts over SSH. We're always interested in suggestions to improve 'go' - so please let us know through the github issues page if you have something you'd like included. We can't promise we'll include every request - but they'll all get looked at :)
